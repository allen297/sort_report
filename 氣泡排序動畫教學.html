📚 介紹

氣泡排序法是一種簡單的排序演演算法，通過重複比較相鄰的元素並交換來工作。 較小的元素像「氣泡」一樣逐漸浮動到列表的頂部，因此得名。

🔍 基本原理

✨ 核心思想

  1. 從左邊開始，比較相鄰的兩個元素
  2. 如果左邊元素大於右邊元素，交換它們
  3. 向右移動並重複比較
  4. 每一輪結束後，最大元素移到末尾
  5. 重複上述過程，直到列表排序完成

視覺化示例

初始列表：[5, 3, 8, 4, 2] 第一輪： [5, 3, 8, 4, 2] → [3, 5, 8, 4, 2] (5 > 3，交換) [3, 5, 8, 4, 2] → [3, 5, 8, 4, 2] (5 < 8，無需交換) [3, 5, 8, 4, 2] → [3, 5, 4, 8, 2] (8 > 4，交換) [3, 5, 4, 8, 2] → [3, 5, 4, 2, 8] (8 > 2，交換) ✓ 第二輪： [3, 5, 4, 2, 8] → [3, 5, 4, 2, 8] (3 < 5，無需交換) [3, 5, 4, 2, 8] → [3, 4, 5, 2, 8] (5 > 4，交換) [3, 4, 5, 2, 8] → [3, 4, 2, 5, 8] (5 > 2，交換) ✓ ...繼續進行... 最終排序結果：[2, 3, 4, 5, 8]

📖 演演算法詳解

偽代碼

procedure bubbleSort(A : list) n := length(A) for i := 0 to n-1 do for j := 0 to n-i-2 do if A[j] > A[j+1] then swap(A[j], A[j+1]) end if end for end for end procedure

逐步執行

1. 初始化： 設定外層迴圈，從 i=0 到 n-1
2. 內層迴圈： 設定 j=0，比較 A[j] 和 A[j+1]
3. 比較交換： 若 A[j] > A[j+1]，交換兩個元素
4. 移動指針： j 向右移動一位
5. 重複： 內層迴圈完成一輪後，最大元素到達末尾
6. 繼續： 外層迴圈繼續，已排序部分減少一位
7. 終止： 外層迴圈完成，列表排序完成

📊 複雜度分析

時間複雜度

情況 複雜度 說明
最好情況 O(n) 列表已排序，通過優化版本可一次遍歷完成
平均情況 O(n²) 列表隨機排列
最壞情況 O(n²) 列表逆序排列，需要最多交換

空間複雜度

指標 複雜度 說明
空間複雜度 O(1) 只使用常數個變數，原地排序

比較次數

對於長度為 n 的列表，總比較次數為：

(n-1) + (n-2) + ... + 1 = n(n-1)/2 ≈ O(n²)

💻 代碼實現

基礎版本

def bubble_sort(arr): n = len(arr) for i in range(n): for j in range(0, n - i - 1): if arr[j] > arr[j + 1]: arr[j], arr[j + 1] = arr[j + 1], arr[j] return arr

優化版本（提早終止）

def bubble_sort_optimized(arr): n = len(arr) for i in range(n): swapped = False for j in range(0, n - i - 1): if arr[j] > arr[j + 1]: arr[j], arr[j + 1] = arr[j + 1], arr[j] swapped = True if not swapped: break return arr

⚖️ 優缺點

✅ 優點

  • 易於理解： 演演算法邏輯簡單直觀，易於初學者學習
  • 原地排序： 不需要額外的存儲空間，空間複雜度為 O(1)
  • 穩定排序： 保持相等元素的相對順序
  • 適應性好： 對於接近排序的列表性能較好

❌ 缺點

  • 效率低下： 時間複雜度為 O(n²)，對大型列表不實用
  • 不適合大數據： 隨著數據量增加，性能急劇下降
  • 有更好替代方案： 快速排序、合併排序等更高效
