📚 介紹

選擇排序法是一種簡單直觀的排序演演算法。它通過每次從未排序的部分選擇最小元素， 然後將其放在已排序部分的末尾。雖然時間複雜度為 O(n²)，但由於其簡單性和穩定性， 在教學和小規模數據排序中仍然很有用。

🔍 基本原理

✨ 核心思想

  1. 將列表分為已排序和未排序兩部分
  2. 在未排序部分中查找最小元素
  3. 將最小元素與未排序部分的第一個元素交換
  4. 已排序部分向右擴展一位
  5. 重複上述過程，直到整個列表排序完成

視覺化示例

初始列表：[5, 3, 8, 4, 2] 第一輪：尋找最小值 2 [5, 3, 8, 4, 2] → [2, 3, 8, 4, 5] (交換 5 和 2) ✓ 第二輪：在 [3, 8, 4, 5] 中尋找最小值 3 [2, 3, 8, 4, 5] → [2, 3, 8, 4, 5] (3 已在正確位置) ✓ 第三輪：在 [8, 4, 5] 中尋找最小值 4 [2, 3, 8, 4, 5] → [2, 3, 4, 8, 5] (交換 8 和 4) ✓ 第四輪：在 [8, 5] 中尋找最小值 5 [2, 3, 4, 8, 5] → [2, 3, 4, 5, 8] (交換 8 和 5) ✓ 最終排序結果：[2, 3, 4, 5, 8]

📖 演演算法詳解

偽代碼

procedure selectionSort(A : list) n := length(A) for i := 0 to n-2 do minIndex := i for j := i+1 to n-1 do if A[j] < A[minIndex] then minIndex := j end if end for if minIndex ≠ i then swap(A[i], A[minIndex]) end if end for end procedure

逐步執行

1. 初始化： 設定外層迴圈，從 i=0 到 n-2
2. 尋找最小值： 設定 minIndex=i，準備查找最小元素
3. 搜尋最小： 內層迴圈從 j=i+1 開始，比較 A[j] 與 A[minIndex]
4. 更新最小值索引： 若 A[j] < A[minIndex]，更新 minIndex
5. 交換元素： 若 minIndex ≠ i，交換 A[i] 和 A[minIndex]
6. 擴展排序部分： 外層迴圈繼續，已排序部分向右擴展
7. 終止： 外層迴圈完成，列表排序完成

📊 複雜度分析

時間複雜度

情況 複雜度 說明
最好情況 O(n²) 即使列表已排序，仍需進行 n(n-1)/2 次比較
平均情況 O(n²) 列表隨機排列
最壞情況 O(n²) 列表逆序排列

空間複雜度

指標 複雜度 說明
空間複雜度 O(1) 只使用常數個變數，原地排序

比較次數

對於長度為 n 的列表，總比較次數固定為：

(n-1) + (n-2) + ... + 1 = n(n-1)/2 ≈ O(n²)

基礎版本

def selection_sort(arr): n = len(arr) for i in range(n - 1): min_index = i for j in range(i + 1, n): if arr[j] < arr[min_index]: min_index = j if min_index != i: arr[i], arr[min_index] = arr[min_index], arr[i] return arr

⚖️ 優缺點

✅ 優點

  • 易於理解： 邏輯直觀簡單，易於初學者理解和實現
  • 原地排序： 不需要額外的存儲空間，空間複雜度為 O(1)
  • 交換次數少： 最多只進行 n-1 次交換，優於氣泡排序
  • 性能穩定： 無論數據如何排列，比較次數都是 O(n²)
  • 實用性： 對於小規模數據排序非常有效

❌ 缺點

  • 時間複雜度高： O(n²) 複雜度，對大型數據集效率低下
  • 不穩定排序： 交換操作可能改變相等元素的相對順序
  • 無優化空間： 無法根據數據特性進行優化（如已排序數據）
  • 有更優方案： 快速排序、歸併排序等提供更好的性能

🔄 與其他排序算法的比較

算法 最好情況 平均情況 最壞情況 空間 穩定性
選擇排序 O(n²) O(n²) O(n²) O(1) ❌ 不穩定
氣泡排序 O(n) O(n²) O(n²) O(1) ✅ 穩定
快速排序 O(n log n) O(n log n) O(n²) O(log n) ❌ 不穩定
歸併排序 O(n log n) O(n log n) O(n log n) O(n) ✅ 穩定

🎓 關鍵概念

為什麼選擇排序不穩定？

考慮這個例子：[(5, A), (3, B), (5, C), (1, D)]
第一輪：尋找最小值 (1, D)，交換到第 0 位
結果：[(1, D), (3, B), (5, C), (5, A)]

在這個過程中，兩個值為 5 的元素的相對順序發生了改變： 原本 (5, A) 在 (5, C) 前面，現在 (5, C) 在 (5, A) 前面。

比較 vs 交換

指標 選擇排序 說明
比較次數 n(n-1)/2 總是固定的，無法優化
交換次數 最多 n-1 遠少於氣泡排序，最多只交換 n-1 次
